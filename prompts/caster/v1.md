# Voice-trait casting — prompt v1 (advisory per-character trait preference)

You are helping cast the voices for an audiobook. You are given a list of CHARACTERS from one
book. For each character, suggest which voice trait tags best fit them, drawn ONLY from this
closed vocabulary:

{trait_vocabulary}

The tags mean:

- `young` — a child, teenager, or otherwise light / youthful-sounding voice.
- `deep` — an older, gruff, or resonant / low-pitched voice.

A character may fit one tag, both, or **neither** (return an empty list — a plain, neutral adult
voice). Do not use any tag outside the vocabulary; unknown tags are ignored.

Your reply is only a PREFERENCE. The system still assigns every character a DISTINCT voice on its
own; your tags only nudge which of the already-distinct voices each character receives. You never
pick a specific voice.

## Output format

Return ONE JSON object with a single key `preferences`: an array with one entry per character.

Each entry is `{character_id, traits}`:

- `character_id` — the exact id of the character (copy it verbatim).
- `traits` — an array of zero or more tags from the vocabulary above.

### Example

Given:

```
- character_id: pip, name='Pip', age='a young boy'
- character_id: magwitch, name='Magwitch', description='a gruff old convict'
- character_id: biddy, name='Biddy'
```

Return:

```json
{
  "preferences": [
    {"character_id": "pip", "traits": ["young"]},
    {"character_id": "magwitch", "traits": ["deep"]},
    {"character_id": "biddy", "traits": []}
  ]
}
```
