# Alias adjudication ‚Äî prompt v1 (approve/reject character-merge pairs)

You are given a short list of CANDIDATE PAIRS of character records from ONE book. For each
pair, decide a single question: **are A and B the same character in this book?**

You do NOT propose merges, rename anyone, or introduce new pairs ‚Äî you only judge the pairs
given, each identified by its `pair_id`.

## Precision comes first

Over-merging two DIFFERENT characters is the worst possible mistake. When the evidence is
not clearly consistent, answer `same_person: false`.

- **Family members are NOT the same person.** Two people who share a surname (siblings, a
  parent and child, a married couple) are distinct characters. `Mr. Bennet` and
  `Mrs. Bennet` are two people; so are `Elizabeth Bennet` and `Jane Bennet`.
- A **bare first name** matches a **full name** only when the evidence (gender, age,
  description, aliases) is consistent and nothing points to a different person of that name.
- A **title + surname** (`Miss Bennet`) matches a **given name + surname** (`Elizabeth
  Bennet`) only when the title fits that person and no sibling ambiguity remains.
- A **nickname/diminutive** (`Lizzy` for `Elizabeth`) matches only when the surrounding
  evidence agrees.
- If two candidates could each be a *different* real person, answer `false`.

## Output format

Return ONE JSON object with a single key `verdicts`: an array with one entry per pair.

Each entry is `{pair_id, same_person, confidence, justification}`:

- `pair_id` ‚Äî the exact id of the pair you are judging (copy it verbatim).
- `same_person` ‚Äî `true` only if A and B are the same character; otherwise `false`.
- `confidence` ‚Äî 0.0‚Äì1.0, how sure you are. Use a high value ONLY when the identification is
  unambiguous; use a low value when it is merely plausible.
- `justification` ‚Äî one short sentence citing the evidence.

### Example

Given:

```
- pair_id: elizabeth::elizabeth_bennet
  A: canonical_name='Elizabeth', gender='female'
  B: canonical_name='Elizabeth Bennet', gender='female', description='the second Bennet daughter'
- pair_id: jane_bennet::miss_bennet
  A: canonical_name='Jane Bennet', gender='female'
  B: canonical_name='Miss Bennet', gender='female'
```

Return:

```json
{
  "verdicts": [
    {"pair_id": "elizabeth::elizabeth_bennet", "same_person": true, "confidence": 0.95,
     "justification": "Same given name and gender; 'Elizabeth' is the bare first name of Elizabeth Bennet."},
    {"pair_id": "jane_bennet::miss_bennet", "same_person": true, "confidence": 0.6,
     "justification": "Plausibly the eldest Miss Bennet, but a sister could also be 'Miss Bennet'."}
  ]
}
```
