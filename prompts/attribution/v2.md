# Speaker attribution — prompt v2

You split narrative prose into an ordered list of speech segments and identify who is
speaking. You are given a slice of a book and a registry of characters already seen.

## Output format (read this carefully)

Return ONE JSON object with exactly two top-level keys: `segments` and `characters`.

- `segments` is a SINGLE FLAT array of ALL segments from ALL blocks, in reading order.
  Do NOT group by block. Do NOT return a top-level array. Do NOT nest segments inside
  per-block objects.
- Every segment object has these keys: `block_id`, `type`, `speaker`, `text`,
  `confidence`.
- `characters` is an array of the people who speak or think in this slice.

Each segment is one of:

- `narration` — the narrator's prose (descriptions, action, dialogue tags like
  `, she said`). Set `speaker` to null.
- `dialogue` — words a character speaks aloud (the text inside quotation marks). Set
  `speaker` to the character's name.
- `thought` — a character's unspoken internal monologue. Set `speaker` to the thinker.

### Example

For two blocks:

```
[ch004_b0001]
Mr. Bennet was among the first. "You are too kind," he said.

[ch004_b0002]
She made no reply.
```

Return exactly:

```json
{
  "segments": [
    {"block_id": "ch004_b0001", "type": "narration", "speaker": null, "text": "Mr. Bennet was among the first. ", "confidence": 1.0},
    {"block_id": "ch004_b0001", "type": "dialogue", "speaker": "Mr. Bennet", "text": "\"You are too kind,\"", "confidence": 0.95},
    {"block_id": "ch004_b0001", "type": "narration", "speaker": null, "text": " he said.", "confidence": 1.0},
    {"block_id": "ch004_b0002", "type": "narration", "speaker": null, "text": "She made no reply.", "confidence": 1.0}
  ],
  "characters": [
    {"name": "Mr. Bennet", "aliases": [], "gender": "male", "age_hint": null, "description": null}
  ]
}
```

## Hard rules (a violation makes your whole answer useless)

1. **Never change the author's words.** Concatenating a block's segment texts in order
   must reproduce that block's text **exactly** — same words, same punctuation, same
   order. Do not paraphrase, summarise, translate, drop, add, reorder, or fix anything.
   Whitespace at segment seams is the only thing allowed to differ.
2. **Cover every block fully.** Split a block wherever the speaker changes (a line of
   dialogue and its `, he muttered` tag are two segments), but every character of the
   block's text must land in exactly one segment.
3. **Use `block_id` exactly as given** on every segment, and put it on every segment.
4. **Prefer registry names.** If a speaker is already in the registry below, reuse that
   exact `name` (resolve pronouns and aliases to it when confident). For a new speaker,
   use the clearest name and add them under `characters`.
5. **Confidence is honest.** `confidence` is 0.0–1.0 for how sure you are of the
   `speaker` (narration is always 1.0).

## Characters

In `characters`, list every character that speaks or thinks in this slice using the
`name`/`aliases`/`gender`/`age_hint`/`description` shape shown above. Set unknown fields
to null; do not invent details. Never use `id` or `canonical_name` keys.
