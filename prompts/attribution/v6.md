# Speaker attribution — prompt v6 (per-quote speaker + emotion + thought candidates)

You read a slice of a book and name who speaks each line of dialogue, and with what emotion.
You also confirm which pre-identified italic passages are genuine interior monologue (a
character's unspoken thought) and name the thinker. You do NOT rewrite, split, or repeat any
text — the system handles that. Your jobs are attribution, emotion, and confirmation.

## Output format

Return ONE JSON object with three top-level keys: `blocks`, `characters`, and `thoughts`.

`blocks` is an array with one entry per block you are given. Each entry has a `block_id` and
attributes that block's dialogue in ONE of two ways:

- **Single-quote (or no quote) block** — the common case. Fill `speaker`:
  - `speaker` is the name of the character who speaks the quoted dialogue in that block. If
    the block has no dialogue (pure narration / description / action), set `speaker` to null.
  - `confidence` is 0.0–1.0 for how sure you are (use 1.0 for null).
  - `emotion` is the emotion of that dialogue (see "Emotion" below), or null for narration.

- **Multi-quote block** — a block whose quoted lines are pre-marked with `⟦Q0⟧`, `⟦Q1⟧`, … .
  These markers are inserted by the system so you can refer to each quote by its number; they
  are NOT part of the text. Fill `quotes` (leave `speaker` null):
  - `quotes` is an array of `{index, speaker, confidence, emotion}`, ONE entry per `⟦Qn⟧`
    marker, where `index` is the marker's number (`⟦Q0⟧` → `index` 0).
  - `speaker` is who speaks THAT quote (null if you cannot tell — it will be treated as
    narration, never guessed).
  - `emotion` is that quote's emotion.

`characters` lists the people who speak or think in this slice, each as
`{name, aliases, gender, age_hint, description}`. Set unknown fields to null; never use
`id` or `canonical_name` keys. Invent nothing.

`thoughts` is an array of `{candidate_id, is_thought, thinker, confidence}`, one entry per
thought candidate listed below (see "Thought candidates").

## Emotion

Each `emotion` is `{label, intensity}` describing HOW a line is spoken:

- `label` is exactly one of: `neutral`, `happy`, `sad`, `angry`, `fearful`, `tender`, `tense`.
  Use `neutral` when no emotion clearly dominates. Only dialogue carries emotion; narration
  gets `null`.
- `intensity` is `1` (mild), `2` (moderate), or `3` (strong).

## Rules

1. **One entry per block**, using each block's exact `block_id`.
2. **Use `quotes` only for a block that has `⟦Qn⟧` markers**; use `speaker` for every other
   block. Return one `quotes` entry per marker, with its exact `index`.
3. **Resolve to who is actually speaking** — follow the dialogue tags (`said Mr. Bennet`),
   pronouns, and the back-and-forth of a conversation. The speaker is rarely the person being
   addressed.
4. **Prefer registry names.** If a speaker is already in the registry below, reuse that exact
   `name`. Add new speakers under `characters`.
5. **Null for narration.** A block or quote with no attributable speaker gets `null`.

## Thought candidates

Each candidate below is an italicized passage the system already located. For EACH one,
return a `thoughts` entry with its exact `candidate_id`:

- Set `is_thought` to `true` ONLY if the passage is a character's genuine unspoken interior
  monologue (a silent thought). Set it to `false` if the italics are mere emphasis, a book /
  song / ship title, a foreign phrase, a letter or sign being read, or anything spoken
  aloud. When unsure, prefer `false`.
- When `is_thought` is `true`, set `thinker` to the name of the character doing the
  thinking (reuse a registry name where possible). Cue verbs nearby — *thought*, *wondered*,
  *mused*, *asked herself* — are good evidence for the thinker but are not required. If you
  cannot name the thinker, set `is_thought` to `false`.
- `confidence` is 0.0–1.0. Never invent a candidate_id that is not listed; only return
  verdicts for the ids given.
