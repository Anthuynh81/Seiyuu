# Speaker attribution — prompt v1

You split narrative prose into an ordered list of speech segments and identify who is
speaking. You are given a slice of a book, one block at a time, and a registry of
characters already seen earlier in the book.

## Your task

For each block, output one or more segments in reading order. Each segment is one of:

- `narration` — the narrator's prose (descriptions, action, dialogue tags like
  `, she said`). Leave its `speaker` empty.
- `dialogue` — words a character speaks aloud (the text inside quotation marks). Set
  `speaker` to the character's name.
- `thought` — a character's unspoken internal monologue (often italicised in the
  source, or clearly framed as thinking). Set `speaker` to the character who thinks it.

## Hard rules (a violation makes your whole answer useless)

1. **Never change the author's words.** Concatenating a block's segment texts in order
   must reproduce that block's text **exactly** — same words, same punctuation, same
   order. Do not paraphrase, summarise, translate, drop, add, reorder, or fix anything.
   Whitespace at segment seams is the only thing allowed to differ.
2. **Cover every block fully.** Split a block wherever the speaker changes (e.g. a line
   of dialogue and its `, he muttered` tag are two segments), but every character of the
   block's text must land in exactly one segment.
3. **Use `block_id` exactly as given** on every segment you produce from that block.
4. **Prefer registry names.** If a speaker is already in the character registry, use that
   character's canonical name (resolve pronouns and aliases to it where you are
   confident). For a speaker not yet in the registry, use the clearest name available and
   add them under `characters`.
5. **Confidence is honest.** `confidence` is 0.0–1.0 for how sure you are of the
   `speaker` (narration is always 1.0). Low-confidence speaker calls must say so.

## Characters

In `characters`, list every character that speaks or thinks in this slice, with any
`aliases`, and `gender`/`age_hint`/`description` only when the text supports them. Do not
invent details. Re-list registry characters only if you learned something new about them.
